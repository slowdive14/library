 동작 방식:

  | 상황                          | 결과                        |
  |-------------------------------|-----------------------------|
  | 새 책 추가 + 이미 대출가능(Y) | ✅ 알림 (기본값 N→Y로 인식) |
  | 대출가능(Y) → 대출가능(Y)     | ❌ 알림 없음                |
  | 대출가능(Y) → 대출중(N)       | ❌ 알림 없음 (상태만 저장)  |
  | 대출중(N) → 대출가능(Y)       | ✅ 알림                     |

  즉:
  - Google Sheet에 새 책을 추가하면, 이미 대출 가능해도 첫 알림은 받음
  - 이후 계속 대출 가능 상태면 알림 없음
  - 누군가 빌렸다가 반납하면 다시 알림


> 새책 추가했는데 대출 불가면 ?

● | 상황                      | 결과               |
  |---------------------------|--------------------|
  | 새 책 추가 + 대출불가(N)  | ❌ 알림 없음 (N→N) |
  | 이후 대출가능(Y)으로 변경 | ✅ 알림 (N→Y)      |

  의도한 동작입니다 - 대출 가능해질 때까지 기다렸다가 알림을 보냅니다.

> 왜 이렇게 의도했어? 로직이 잘 이해가 안 돼 쉽게 설명

● 목적: "빌리고 싶은 책이 반납되면 알려줘"

  시나리오로 설명:

  1. 내가 "코스모스"를 빌리고 싶어서 Sheet에 추가함
  2. 지금 누가 빌려서 대출중(N) → 알림 필요 없음 (어차피 못 빌림)
  3. 며칠 후 반납됨 → 대출가능(Y) → 이때 알림! 🔔

  반대 상황:

  1. "트렌드 코리아"를 Sheet에 추가함
  2. 지금 대출가능(Y) → 바로 알림! 🔔 (지금 빌릴 수 있으니까)

  ---
  핵심: "지금 빌릴 수 있어!"라는 순간에만 알림

  - 이미 빌릴 수 있으면 → 바로 알려줌
  - 지금 못 빌리면 → 빌릴 수 있게 될 때 알려줌
  - 계속 빌릴 수 있으면 → 매번 알릴 필요 없음
