# 문제
> 텔레그램에서 명령어를 입력해도 봇이 아무런 응답을 하지 않음.

---

# 시도
- `python bot.py` 실행 시도 (실패: `ModuleNotFoundError`)
- `Get-Process`로 실행 중인 Python 프로세스 확인 (기존 프로세스 존재 확인)
- 절대 경로 Python(`C:\Users\user\AppData\Local\Python\bin\python.exe`)으로 실행 시도 (실패: `gspread` 모듈 없음)

---

# 원인
- **의존성 미설치**: 시스템의 특정 Python 환경에 `gspread` 등 필수 라이브러리가 설치되어 있지 않았음.
- **프로세스 중복**: 기존에 실행 중이던 다른 봇 프로세스가 텔레그램 서버의 업데이트(`getUpdates`)를 선점하여 새로운 프로세스가 응답을 받지 못함.

---

# 해결
- `requirements.txt`에 명시된 모든 라이브러리를 올바른 Python 환경에 설치: `& "C:\Users\user\AppData\Local\Python\bin\python.exe" -m pip install -r requirements.txt`
- 충돌하는 기존 Python 프로세스 강제 종료: `Stop-Process -Id 7888`
- 봇 재실행 및 텔레그램 서버 통신(`200 OK`) 확인

# 재발 방지
- 봇 실행 시 항상 환경이 검증된 절대 경로의 Python 실행 파일을 사용하거나 가상환경(venv)을 활성화할 것.
- 봇이 먹통일 경우 `Get-Process` 명령어로 중복 실행 여부를 먼저 확인할 것.

# 📝 쉬운 설명

### 문제
텔레그램 봇에게 말을 시켜도 대답을 안 하는 문제가 있었습니다.

### 원인  
봇이 작동하는 데 꼭 필요한 '부품'(gspread 라이브러리)이 빠져 있었고, 똑같은 봇이 다른 곳에서 이미 켜져 있어서 대화 내용을 가로채고 있었습니다.

### 해결
빠진 부품들을 모두 채워 넣고, 이미 켜져 있던 봇을 끈 다음 다시 실행해서 봇이 대화에 집중할 수 있게 고쳤습니다.
